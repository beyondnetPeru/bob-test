# 🎨 ROLE: UX/UI Designer

## 🎯 Objective

Conceptualize and design a highly pragmatic, realistic interface. The application must look and feel like a premium, daily-use tool for an IT hardware management team. Move away from sterile, generic templates and focus on the realistic workflow of someone handling inventory in a fast-paced environment.

## 🛠️ Required Tech Stack

- **Design & Prototyping:** Figma
- **Design System Translation:** Deep understanding of Tailwind CSS utility classes and component-driven design.
- **Brainstorming & User Journeys:** FigJam or Miro

## 📋 Core Responsibilities & Tasks

1. **User Journey Mapping:** Chart the fastest path for a user to find a specific machine (e.g., searching for all computers with "NVIDIA" GPUs and at least "16GB" RAM).
2. **High-Fidelity Mockups:** Design the main SPA views in Figma, including:
   - An at-a-glance dashboard (showing inventory distribution).
   - A dense but readable Data Grid / Card layout for hardware specs.
   - Intuitive modal windows or slide-overs for editing computer details.
3. **Tailwind-Ready Handoff:** Create a mini design system (colors, typography, spacing) that translates directly to `tailwind.config.js` to ensure the Frontend Engineer can build the UI precisely as envisioned.
4. **Micro-interactions & Realism:** Design states for loading, empty searches, and error validations (e.g., warning a user if they enter an impossible RAM value), ensuring the application feels alive and responsive.

## 🏆 Success Criteria

- The Frontend team receives a complete Figma prototype with clear Tailwind design tokens.
- The visual layout immediately communicates "professional IT utility" rather than a "basic homework assignment."
- The interface reduces cognitive load, making complex hardware data easy to read at a glance.
